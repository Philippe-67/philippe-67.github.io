Checkout

Définition : La commande git checkout permet de se positionner sur une branche ou un commit spécifique.

Exemple : git checkout

Branch

Définition : La commande git branch est utilisée pour créer une nouvelle branche.

Exemple : git branch

Merge

Définition : La commande git merge permet de fusionner une branche dans la branche courante.

Exemple : git merge

Rebase

Définition : La commande git rebase permet de déplacer une branche sur un autre commit.

Exemple : git rebase

Détacher HEAD

Définition : Détacher HEAD signifie pointer HEAD vers un commit spécifique sans être sur une branche.

Exemple : git checkout

Références Relatives

Définition : Les références relatives permettent de se déplacer dans l'historique des commits.

Exemples :

  • git checkout main^
  • git checkout HEAD~4

Forcer les Branches

Définition : On peut réassigner une branche à un commit spécifique avec l'option -f.

Exemple : git branch -f main HEAD~3

Annuler des Changements

Définition : La commande git reset permet d'annuler des changements.

Exemples :

  • git reset HEAD~1
  • git revert

Hashes

Définition : Les hashes sont des identifiants uniques pour chaque commit.

Exemple : Utilisez le hash pour identifier un commit : git checkout

Conclusion

Ces commandes Git sont essentielles pour la gestion de votre code. En les maîtrisant, vous serez en mesure de manipuler efficacement votre historique de code et de collaborer avec d'autres développeurs.
